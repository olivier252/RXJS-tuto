Angular et un framework Javascript complet qui permet de développer une application web dynamique.

Ce que l'on peut faire avec Angular

  • - Single Page Application (SPA) : le navigateur du client exécute le code JS et fait tourner l'application de façon dynamique, et ce, sur une seule page web.
  • - Développer des applications hybrides pour smartphone avec Ionic 4 ou encore Angular for NativeScript.
  • - Progressive Web App (PWA) : applications de bureau ou smartphone qui se basent sur une URL pour être installées.
  • - Créer des composants web qui peuvent être utilisés dans n'importe quelle techno front.

Les technologies Angular

  • - Angular Universal et le Sever Side Rendering.
  • - Angular elements, les composants web réutilisables.
  • - Le Lazyloading : une partie d'un projet seulement se charge quand l'utilisateur veut y accéder, indispensable pour réduire le temps de chargement.
  • - Modules ou packages Angular qui fournissent une boite à outil très complète pour le développeur. Les principaux : formulaires, routing, HTTP, tests…
  • - Angular-CLI : permet d'utiliser pléthore de commandes pour optimiser le développement.
  • - Une documentation très fournie : angular.io

En JS un objet n'est pas forcement une instance de classe ! Les interfaces sont des contrats. Une interface définit ce qui se trouve à l’intérieur d’un objet (encore une fois… pas une instance d’une classe). Lorsque vous définissez votre interface Vous dites que tout objet donné dans ce contrat doit être un objet contenant 2 propriétés qui sont spécifiquement appelées « a » et « b » et qui sont respectivement un nombre et une chaîne. Ce qui se passe, c’est que typescript va alors lancer une erreur à chaque fois que vous ne respectez pas ce contrat (par exemple si un paramètre d’une fonction est censé être un MyInterface, vous ne pouvez rien passer d’autre).

Le JavaScript traditionnel utilise des fonctions et un héritage basé sur un prototype pour construire des composants réutilisables, mais cela peut sembler un peu gênant pour les programmeurs plus à l’aise avec une approche orientée objet, où les classes héritent des fonctionnalités et les objets sont construits à partir de ces classes.
